┌─────────────────────────────────────────────────────────────────────────────┐
│                           TRAINING LOOP COMPARISON                          │
└─────────────────────────────────────────────────────────────────────────────┘

CTGAN (Simple):
┌────────────────────────────────────────┐
│  For each epoch:                       │
│    For each batch:                     │
│      1. Train Discriminator (1x)      │
│         └─> Update weights             │
│      2. Train Generator (1x)           │
│         └─> Update weights             │
│                                        │
│  Total: 2 updates per batch            │
└────────────────────────────────────────┘

PATE-CTGAN (Complex):
┌────────────────────────────────────────────────────────────────┐
│  While privacy_budget < epsilon:                               │
│    1. Train N=10 Teacher Discriminators                        │
│       └─> teacher_iters=5 × N=10 teachers = 50 updates        │
│                                                                │
│    2. Train Student Discriminator                             │
│       ├─> Aggregate teacher predictions (PATE)                │
│       ├─> Add Laplacian noise                                 │
│       └─> student_iters=5 updates                             │
│                                                                │
│    3. Train Generator (1x)                                    │
│       └─> Update weights                                      │
│                                                                │
│    4. Track Privacy Budget                                    │
│       └─> Update epsilon, check threshold                     │
│                                                                │
│  Total: 56+ updates per iteration                             │
└────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                         WHY IS PATE-CTGAN SLOWER?                           │
└─────────────────────────────────────────────────────────────────────────────┘

1. Multiple Discriminators (11x overhead)
   ├─ CTGAN: 1 discriminator
   └─ PATE-CTGAN: 10 teachers + 1 student = 11 total

2. Extra Training Phases (3x vs 2x)
   ├─ CTGAN: Discriminator → Generator
   └─ PATE-CTGAN: Teachers → Student → Generator

3. More Updates per Iteration (28x more)
   ├─ CTGAN: 1 discriminator + 1 generator = 2 updates
   └─ PATE-CTGAN: 50 teacher + 5 student + 1 generator = 56 updates

4. PATE Aggregation Overhead
   ├─ Collect predictions from 10 teachers
   ├─ Aggregate votes
   ├─ Add Laplacian noise
   └─ Compute consensus

5. Privacy Budget Tracking
   ├─ Moments accountant calculation
   ├─ Epsilon tracking
   └─ Early stopping logic

6. Data Partitioning
   ├─ Split dataset into N partitions
   └─ Each teacher trains on subset

7. Smaller Batch Size
   ├─ CTGAN: batch_size=200
   └─ PATE-CTGAN: batch_size=100 (slower convergence)

Reasons:
  • Bean dataset is public ML benchmark (not sensitive)
  • No privacy requirements
  • 2x faster training (25 min vs 50 min)
  • Simpler debugging and hyperparameter tuning
  • Quality difference minimal for non-private data

┌─────────────────────────────────────────────────────────────────────────────┐
│                      PERFORMANCE OPTIMIZATION TIPS                          │
└─────────────────────────────────────────────────────────────────────────────┘

To Speed Up PATE-CTGAN:
  1. Reduce num_teachers: 10 → 5 (2x speedup)
  2. Reduce teacher_iters: 5 → 3 (1.6x speedup)
  3. Reduce student_iters: 5 → 3 (1.6x speedup)
  4. Increase batch_size: 100 → 200 (faster convergence)
  5. Parallelize teachers across multiple GPUs

  Combined potential speedup: 3-4x
  Trade-off: May reduce privacy guarantees
